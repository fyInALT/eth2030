# US-10.1 — Cross-Frame State Interactions

**Epic:** EP-10 Frame Interactions & Cross-Frame State
**Total Story Points:** 5
**Sprint:** 4

> **As a** protocol engineer,
> **I want** the access list warm/cold state journal to persist across all frames in a transaction,
> **so that** gas accounting for repeated state accesses is consistent within a transaction.

**INVEST:** I ✓ | N ✓ | V ✓ | E ✓ | S ✓ | T ✓

---

## Tasks

### Task 10.1.1 — Cross-Frame Access List Journal

| Field | Detail |
|-------|--------|
| **Description** | Maintain a single `AccessList` / warm-state journal across all frames. Frame N warms a slot → Frame N+1 sees it warm. Opposite of transient storage. Share state journal in `pkg/core/core/aa_executor.go`. |
| **Estimated Effort** | 3 story points |
| **Assignee/Role** | Core Protocol Engineer |
| **Testing Method** | (1) Frame 0 reads `address(X)` slot → cold (2100 gas). (2) Frame 1 reads same slot → warm (100 gas). (3) Assert gas charged correctly. (4) Applies to accounts as well as storage slots. |
| **Definition of Done** | Tests pass; warm state not reset between frames; gas charges correct; reviewed. |

### Task 10.1.2 — Transient Storage (TSTORE/TLOAD) Isolation Between Frames

| Field | Detail |
|-------|--------|
| **Description** | Transient storage must be cleared between frames. `TSTORE` in Frame N is not visible to Frame N+1. Clear transient storage after each frame completes. |
| **Estimated Effort** | 2 story points |
| **Assignee/Role** | Core Protocol Engineer |
| **Testing Method** | (1) Frame 0 `TSTORE(key, value)`. (2) Frame 1 `TLOAD(key)` → returns 0. (3) Within single frame, TSTORE→TLOAD works normally. (4) Sub-calls within a frame share transient storage. |
| **Definition of Done** | Tests pass; isolation confirmed; sub-call behavior unaffected; reviewed. |
