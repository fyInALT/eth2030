# US-11.1 — ORIGIN Opcode Behavior Change

**Epic:** EP-11 ORIGIN Opcode Behavior Change
**Total Story Points:** 4
**Sprint:** 2

> **As a** protocol engineer,
> **I want** the `ORIGIN` opcode to return the frame's caller (ENTRY_POINT or tx.sender) for frame transactions,
> **so that** contracts that use ORIGIN for identification get a consistent, meaningful value.

**INVEST:** I ✓ | N ✓ | V ✓ | E ✓ | S ✓ | T ✓

## Tasks

### Task 11.1.1 — ORIGIN Opcode Override for Frame Transactions

| Field | Detail |
|-------|--------|
| **Description** | Modify `opOrigin`: when in frame transaction context, return the frame's caller address. DEFAULT/VERIFY → `ENTRY_POINT`. SENDER → `tx.sender`. Override applies throughout **all call depths** (sub-calls also see frame caller as ORIGIN). Check `FrameContext != nil` in EVM context. |
| **Estimated Effort** | 3 story points |
| **Assignee/Role** | EVM Engineer |
| **Testing Method** | (1) DEFAULT frame: `ORIGIN` → `ENTRY_POINT` at any call depth. (2) SENDER frame: `ORIGIN` → `tx.sender`. (3) VERIFY frame: `ORIGIN` → `ENTRY_POINT`. (4) Non-frame tx: unchanged. (5) Regression: existing ORIGIN tests still pass. |
| **Definition of Done** | Tests pass for all modes; non-frame txs unaffected; regression pass; reviewed. |
