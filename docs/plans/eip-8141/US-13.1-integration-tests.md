### Task 13.1.4 — Negative Test Cases

| Field | Detail |
|-------|--------|
| **Description** | Tests for all invalid scenarios: (1) VERIFY without APPROVE → invalid tx; (2) SENDER without sender_approved → invalid tx; (3) payer_approved never set → invalid tx; (4) double APPROVE(0x0) → revert; (5) APPROVE from non-target → revert; (6) insufficient balance → revert; (7) static constraint violations → rejected before execution. |
| **Estimated Effort** | 5 story points |
| **Assignee/Role** | QA Engineer |

| **Definition of Done** | All 7 scenarios tested; pass/fail distinction correct; reviewed. |
